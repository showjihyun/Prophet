MiroFish의 핵심 엔진인 **OASIS(Open Agent Social Interaction Simulations)**는 CAMEL-AI 오픈소스 커뮤니티에서 개발한 **초거대규모 사회적 상호작용 시뮬레이터**입니다. 

MiroFish가 단순히 '예측'을 넘어서 '사회적 현상'을 재현할 수 있는 이유는 바로 이 OASIS의 구조 덕분입니다. OASIS에 대해 5가지 핵심 요소로 나누어 자세히 분석해 드립니다.

---

## 1. OASIS의 핵심 구조 (5대 컴포넌트)
OASIS는 단순히 LLM을 여러 개 띄우는 것이 아니라, 실제 소셜 미디어의 작동 원리를 모사하는 5개의 모듈로 구성됩니다.

* **Environment Server (환경 서버):** 에이전트들의 프로필, 관계(팔로우 등), 과거 포스트 이력을 저장하는 중추입니다.
* **Recommendation System (RecSys):** 가장 중요한 부분 중 하나입니다. 실제 SNS처럼 모든 글을 다 보여주는 게 아니라, **알고리즘(관심사 기반, 핫 스코어 등)**에 따라 에이전트가 볼 수 있는 정보를 필터링하여 '확증 편향'이나 '필터 버블' 현상을 유도합니다.
* **Agent Module (에이전트 모듈):** 에이전트의 '뇌'입니다. LLM을 사용하여 인격(Persona)에 맞는 의사결정을 내립니다.
* **Time Engine (시간 엔진):** 현실감을 부여합니다. 에이전트가 24시간 내내 활동하는 게 아니라, 실제 사람처럼 특정 시간대에 접속하고 행동하게 조율합니다.
* **Scalable Inferencer (확장형 추론기):** 수만~수백만 명의 에이전트가 동시에 생각할 때 발생하는 연산 병목을 해결하는 최적화 레이어입니다.

## 2. 주요 기능 및 특징
* **23가지 소셜 액션:** 단순한 댓글 작성을 넘어 팔로우, 리포스트, 좋아요, 뮤트(차단), 검색 등 실제 SNS에서 일어나는 거의 모든 상호작용을 지원합니다.
* **100만 에이전트 확장성:** 논문에 따르면 최대 100만 명 규모의 시뮬레이션이 가능하도록 설계되었습니다. 이는 '여론 형성' 단계를 넘어선 '사회적 거대 담론' 시뮬레이션을 타깃으로 합니다.
* **사회 현상 재현:** 정보 확산(Information Spreading), 집단 극화(Group Polarization), 군중 효과(Herd Effects) 등 사회 심리학적 현상을 통계적으로 유의미하게 복제해낼 수 있습니다.

## 3. MiroFish와의 시너지
MiroFish는 이 OASIS 위에 **GraphRAG**와 **Zep Cloud(장기 기억)**를 얹은 형태입니다.
1.  **입력:** 문서(뉴스, 보고서 등)를 넣으면 GraphRAG가 지식 그래프를 만듭니다.
2.  **생성:** 이 그래프를 기반으로 수천 명의 페르소나를 OASIS 에이전트로 생성합니다.
3.  **실행:** OASIS 엔진이 이들을 Twitter/Reddit 같은 가상 플랫폼에서 싸우고 토론하게 만듭니다.
4.  **결과:** Zep Cloud에 저장된 에이전트들의 기억 변화를 분석해 "이 정책/뉴스가 나오면 미래에 이런 여론이 형성될 것이다"라고 보고서를 씁니다.

## 4. OASIS의 장점과 한계 (Prophet 비교 관점)

| 장점 (Strengths) | 한계 (Limitations) |
| :--- | :--- |
| **검증된 학술적 기반:** CAMEL-AI의 연구 결과물로 사회과학적 타당성이 높음 | **높은 리소스 소모:** 대규모 시뮬레이션 시 LLM API 비용과 연산량이 막대함 |
| **복잡한 인터랙션:** 23종의 액션으로 매우 정교한 커뮤니티 반응 시뮬레이션 가능 | **커스터마이징 난이도:** 내장 엔진이라 특정 비즈니스 로직(예: 특정 앱의 UI/UX 반응) 이식은 어려움 |
| **플랫폼 모사:** Twitter, Reddit 등 매체 특성에 따른 여론 확산 차이를 분석 가능 | **상용화 최적화 미흡:** 연구 목적이 강해 Prophet처럼 3-Tier 비용 절감 같은 실무적 장치가 부족함 |
